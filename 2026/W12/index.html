Diff and History Tabs Land in mdts

The markdown tool now shows uncommitted changes and previous commit diffs, plus a new app listing for gitlogue.

This week was a blend of routine dependency maintenance across multiple personal projects and two significant contributions: a major feature in my markdown editor mdts and a critical bug fix in an external project committers.top. The dependency updates, while numerous, were straightforward, but the feature work required deeper design and implementation thinking.

In mdts, I added two new tabs—Diff and History—to the markdown content view. The Diff tab displays uncommitted working tree changes via git diff, and the History tab shows the previous commit's changes via git diff HEAD~1 HEAD. Both use color-coded rendering for clarity, addressing a long-standing request for better change visibility during editing. Meanwhile, in ashkulz/committers.top, I fixed a user search bug caused by GitHub's GraphQL API not supporting the sort:followers qualifier, switching to the REST API to ensure deterministic, complete rankings.

The week also brought a pleasant side effect: my gitlogue tool was accepted into the awesome-cli-apps list, providing external validation. Overall, it was a productive balance of upkeep and meaningful feature work that enhances developer workflows.
